다음

현재 single_cue 모드에서 서버가 파일을 5개 읽는 구조를 1개(큐 블록만)로 단축. context 토큰 절감 + 추가 속도 향상.

다음

큐 카드 드래그 재배열

카드 순서를 드래그로 변경하면 큐시트·프롬프트 파일 내 블록 순서도 자동 재정렬. 스팟팅 후 순서 조정이 빈번해 필요성 높음.

예정

SUNO 결과 오디오 임베드

SUNO에서 생성한 MP3를 큐 카드에 직접 붙여 재생. A/B/C 변형별 오디오를 탭 전환하며 비교 청취.

예정

화간 일관성 대시보드

음악 바이블의 악기 팔레트·모티프 배치를 화수별로 시각화. 빌런 테마가 몇 화에 몇 번 쓰였는지 한눈에 파악.

예정

멀티 프로젝트 전환 UI

현재는 한 작품(며느리가 친정을 숨김)에 고정. 헤더에서 작품을 드롭다운으로 전환하면 해당 작품 데이터로 전환되는 구조 추가.

아이디어

PDF 대본 자동 추출

제작사에서 받은 PDF 대본을 그대로 업로드하면 OCR + 씬 분리까지 자동 처리해 00_대본raw.txt 생성. 현재는 수동 정리 필요.

아이디어

변형 비교 A/B 테스트 플레이어

같은 씬에서 A·B·C를 순서 없이 무작위 재생, 어떤 변형인지 모르는 상태에서 선택. 선입견 없는 음악 선택 지원.

기술
무엇으로 만들었나
외부 프레임워크 없이 표준 라이브러리 + 바닐라 JS로 구성. 별도 설치 없이 python server.py 한 줄로 실행됩니다.
